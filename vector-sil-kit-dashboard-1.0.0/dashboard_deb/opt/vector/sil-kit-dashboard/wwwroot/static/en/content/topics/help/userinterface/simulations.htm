Simulations

 

The Simulations page is the landing page of SIL Kit Dashboard. It gives an overview of all simulations with their current state and allows the selection of a simulation for a detailed analysis.

The content area of the Simulations page is structured as follows:

[1] Table Header

The table header provides configurable columns in which the information about the simulations is shown. Refer to section Table Data below for information about the available columns.

To configure the columns, click the icon of a column. A menu opens in which you can sort, filter, or show or hide columns.

[2] Table Data

The table data shows the data of the simulations. The columns to be displayed can be configured via the table header.

The following columns are available:

Deleted

The Deleted column is hidden by default. If simulation data has been deleted, a check mark is displayed here, otherwise an empty square. For more information, see Actions column.

S

The S column shows the state of each simulation:

Success The simulation has been processed successfully and shutdown.
Failure An error occurred during the simulation.

If the simulation is in the Running state, a spinner icon is displayed.

#

The # column shows the ID of the simulations.

Connect URI

The Connect URI column shows the SIL Kit Registry URI of the simulations. The search field below the column name can be used to filter for URIs.

Start

The Start column shows when a SIL Kit simulation has been started.

End

The End column shows when a SIL Kit simulation has ended. If a simulation is still running, this field is empty.

System State

The System Status column shows in which state of the state machine the SIL Kit is.

Actions

The Actions column provides the following options:

 Click this icon to open the Network Setup page of the respective simulation.

Click this icon to open the Status History page of the respective simulation.

/

These options are only available for users with Expert role.

Click the icon to delete the respective simulation data. After deletion, the simulation data is hidden, but remains accessible during a defined retention period specified in the application properties of the (missing or bad snippet).

To restore data, the table columns must be configured via the table header as follows:

  1. Activate the check box Columns|Deleted.

  2. Deactivate the check boxes of all Filter.

The simulation data marked to be deleted is displayed again and the icon is available. Click to restore the respective simulation data.

[3] Paging

Click the Items per page arrow to choose how many items per page to be displayed.

To scroll through the pages, use the navigation icons.
