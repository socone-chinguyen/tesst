Network Setup

 

The Network Setup page shows the network setup of a specific simulation from different views, e.g. as an overview, with focus on a specific participant, or with focus on a specific network.

The content area of the Network Setup page is structured as follows:

[1] Selected Simulation

This area displays the key facts about the selected simulation.

[2] Automatic Page Refresh

If the test is in state running, the automatic page refresh is activated by default which is indicated by the icon .

To pause the automatic page refresh, click the icon. To restart the automatic page refresh, click the icon.

[3] View Selection

In the view selection area, you can change the view that is used to display the network setup. To switch the view, click on the current view name and choose from the following views:

  • Overview: Shows an overview of the network setup.

  • Participants: Focuses a selected participant of the network.

  • Networks: Displays a selected network.

  • Topics: Displays an overview of a publish/subscribe communication.

  • Function Names: Displays an overview of an RPC communication.

[4] Network Graph

The Network Graph represents the network setup from the perspective selected in the view selection area.

The outline of each graph element provides information about the state of the respective element, such as warning or error. In case of a warning or an error, the tooltip of the respective graph element provides information about the details.

[5] Tabs

The tabs are used to select the type of table to be displayed. Depending on which view is chosen and which element is selected in the network diagram, the following tabs are available:

  • Networks: Shows information about the selected vehicle network connection (CAN/CAN FD, Ethernet, FlexRay, or LIN).

  • Data / RPC: Shows information about the selected non-real network connection (publish/subscribe or RPC communication).

  • Controllers: Shows information about the services of the selected participant provided via a vehicle network.

  • Data / RPC Services: Shows information about the services provided via publish/subscribe or RPC API by the selected participant.

[6] Table Header

The table header provides configurable columns in which the information about the network setup is shown.

To configure the columns, click the icon of a column. A menu opens in which you can sort or show or hide columns.

[7] Table Data

The table data shows additional information about a connection or participant selected in the network graph. The columns to be displayed can be configured via the table header.

Click on a connection or participant in the network graph to display its details. Depending on the selected tab, the following information is displayed:

Networks

The following columns are available in the Networks table:

  • S: In case of misconfiguration, an icon is displayed here. The tooltip of the icon provides information about the type of misconfiguration, for example if too many network simulators simulate the same network.

  • Network Name: Shows the name of the selected network.

  • Network Type: Shows the network type.

  • Simulated by: Shows how the network is simulated.

Data / RPC

The following columns are available in the Data / RPC table:

  • S: In case of misconfiguration, an icon is displayed here. The tooltip of the icon provides information about the type of misconfiguration.

  • Link Name: Shows the link name of the selected connection.

  • Link Type: Shows the type of service, i.e. Data for publish/subscribe services or RPC for RPC services.

  • Topic / Function Name: Shows the topic name for publish/subscribe services or the function name for RPC services.

Controllers

The following columns are available in the Controllers table:

  • S: In case of misconfiguration, an icon is displayed here, for example, if the participant creates multiple services for a network or no matching network controller was found. The tooltip of the icon provides information about the type of misconfiguration.

  • Service Type: Shows the type of the selected network controller.

  • Name: Shows the name of the selected network controller.

  • Network Name: Shows the name of the network in which the selected network controller is located.

Data / RPC Services

The following columns are available in the Data / RPC Services table:

  • S: In case of a misconfiguration, an icon is displayed here. The tooltip of the icon gives information about the type of misconfiguration.

  • Service Type: Shows the type of services and whether they are provided or received.

  • Topic / Function Name: Shows the topic name for publish/subscribe or the function name for RPC services.

  • Media Type: Displays the media type defined for a publish/subscribe or RPC service.

  • Labels: Displays the labels (key-value pairs) defined for a publish/subscribe or RPC service.

[8] Paging

Click the Items per page arrow to choose how many items per page to be displayed.

To scroll through the pages, use the navigation icons.
