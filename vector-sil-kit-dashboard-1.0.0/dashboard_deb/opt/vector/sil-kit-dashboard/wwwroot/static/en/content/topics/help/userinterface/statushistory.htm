Status History

 

The Status History page shows a historical view of participants statuses for a specific simulation.

The content area of the Status History page is structured as follows:

[1] Selected Simulation

This area displays the key facts about the selected simulation.

[2] Automatic Page Refresh

If the test is in state running, the automatic page refresh is activated by default which is indicated by the icon .

To pause the automatic page refresh, click the icon. To restart the automatic page refresh, click the icon.

[3] Table Header

The table header provides configurable columns in which the information about the status history is shown. Refer to section Table Data below for information about the available columns.

To configure the columns, click the icon of a column. A menu opens in which you can sort, filter, or show or hide columns.

[4] Table Data

The table data shows the data of the participant statuses. The columns to be displayed can be configured via the table header.

The following columns are available:

S

The S column shows the status of each participant. In case of an error, an icon is displayed here.

Participant Name

The Participant Name column shows the name of the participants.

State

The State column shows in which state of the state machine a participant is.

Entry Time

The Entry Time column shows when a participant has changed to the displayed state.

Enter Reason

The Reason column shows why a participant has changed to the displayed state.

Duration

The Duration column shows how long a participant has been in the displayed state.

[5] Paging

Click the Items per page arrow to choose how many items per page to be displayed.

To scroll through the pages, use the navigation icons.
