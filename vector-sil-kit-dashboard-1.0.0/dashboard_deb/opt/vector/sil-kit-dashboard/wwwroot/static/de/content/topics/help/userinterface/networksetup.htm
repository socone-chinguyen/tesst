Netzwerk-Setup

 

Die Seite Netzwerk-Setup zeigt den Netzwerkaufbau einer bestimmten Simulation aus verschiedenen Blickwinkeln an, z. B. als Übersicht, mit Fokus auf einen bestimmten Teilnehmer oder mit Fokus auf ein bestimmtes Netzwerk.

Der Inhaltsbereich der Seite Netzwerk-Setup ist wie folgt aufgebaut:

[1] Ausgewählte Simulation

In diesem Bereich werden die wichtigsten Fakten über die ausgewählte Simulation angezeigt.

[2] Automatische Aktualisierung der Seite

Wenn sich die Simulation im Zustand Läuft befindet, ist die automatische Aktualisierung der Seite standardmäßig aktiviert, was durch das Icon angezeigt wird.

Um die automatische Aktualisierung der Seite anzuhalten, klicken Sie auf das Icon . Um die automatische Aktualisierung der Seite erneut zu starten, klicken Sie auf das Icon .

[3] Auswahl der Ansicht

Im Bereich zur Auswahl der Ansicht können Sie die Ansicht ändern, die zur Anzeige des Netzwerkaufbaus verwendet wird. Um die Ansicht zu wechseln, klicken Sie auf den Namen der gewünschten Ansicht:

  • Übersicht: Zeigt eine Übersicht über den Netzwerkaufbau.

  • Teilnehmer: Fokussiert einen ausgewählten Teilnehmer des Netzwerks.

  • Netzwerke: Zeigt ein ausgewähltes Netzwerk an.

  • Topics: Zeigt eine Übersicht über eine Publish/Subscribe-Kommunikation an.

  • Funktionsnamen: Zeigt eine Übersicht über eine RPC-Kommunikation an.

[4] Netzwerkdiagramm

Das Netzwerkdiagramm stellt den Netzwerkaufbau aus der Perspektive dar, die im Bereich zur Auswahl der Ansicht ausgewählten wurde.

Die Umrandung jedes Grafikelements gibt Auskunft über den Zustand des jeweiligen Elements, z. B. Warnung oder Fehler. Im Falle einer Warnung oder eines Fehlers gibt der Tooltip des jeweiligen Grafikelements Auskunft über die Details.

[5] Registerkarten

Über die Registerkarten können Sie die Art der anzuzeigenden Tabelle auswählen. Abhängig davon, welche Ansicht ausgewählt und welches Element im Netzwerkdiagramm selektiert ist, stehen die folgenden Registerkarten zur Verfügung:

  • Netzwerke: Zeigt Informationen über die selektierte Fahrzeug-Netzwerkverbindung (CAN/CAN FD, Ethernet, FlexRay oder LIN) an.

  • Data / RPC: Zeigt Informationen über die selektierte, nicht reale Netzwerkverbindung (Publish/Subscribe- oder RPC-Kommunikation) an.

  • Controllers: Zeigt Informationen über die über das Fahrzeug-Netzwerk bereitgestellten Services des selektierten Teilnehmers an.

  • Data / RPC Services: Zeigt Informationen über die über Publish/Subscribe oder RPC API bereitgestellten Services des selektierten Teilnehmers an.

[6] Tabellenkopf

Der Tabellenkopf beinhaltet konfigurierbare Spalten, in denen die Informationen über den Netzwerkaufbau angezeigt werden.

Um die Spalten zu konfigurieren, klicken Sie auf das Icon einer Spalte. Es öffnet sich ein Menü, in dem Sie Spalten sortieren oder ein- und ausblenden können.

[7] Tabellendaten

Die Tabellendaten zeigen die Informationen über die TestläufeTestergebnisse an. Die anzuzeigenden Spalten können über den Tabellenkopf konfiguriert werden.Die folgenden Spalten sind verfügbar:Die Tabellendaten zeigen zusätzliche Informationen über die im Netzwerkdiagramm ausgewählte Verbindung oder Teilnehmer an. Die anzuzeigenden Spalten können über den Tabellenkopf konfiguriert werden.

Klicken Sie auf eine Verbindung oder einen Teilnehmer im Netzwerkdiagramm, um dessen Details anzuzeigen. Abhängig von der ausgewählten Registerkarte werden die folgenden Informationen angezeigt:

Netzwerke

In der Tabelle Netzwerke sind die folgenden Spalten verfügbar:

  • S: Im Falle einer Fehlkonfiguration wird hier ein Icon angezeigt. Der Tooltip des Icons gibt Auskunft über die Art der Fehlkonfiguration, beispielsweise wenn zu viele Netzwerksimulatoren dasselbe Netzwerk simulieren.

  • Netzwerkname: Zeigt den Namen des selektierten Netzwerks an.

  • Netzwerktyp: Zeigt den Netzwerktyp an.

  • Simuliert von: Zeigt an, wie das Netzwerk simuliert wird.

Data / RPC

In der Tabelle Data / RPC sind die folgenden Spalten verfügbar:

  • S: Im Falle einer Fehlkonfiguration wird hier ein Icon angezeigt. Der Tooltip des Icons gibt Auskunft über die Art der Fehlkonfiguration.

  • Linkname: Zeigt den Linknamen der ausgewählten Verbindung an.

  • Linktyp: Gibt die Art des Services an, d.h. Data für Publish/Subscribe-Services bzw. RPC für RPC-Services.

  • Topic / Funktionsname: Zeigt den Topicnamen bei Publish/Subscribe- bzw. den Funktionsnamen bei RPC-Services an.

Controllers

In der Tabelle Controllers sind die folgenden Spalten verfügbar:

  • S: Im Falle einer Fehlkonfiguration wird hier ein Icon angezeigt, beispielsweise wenn der Teilnehmer mehrere Services für ein Netzwerk erzeugt oder kein passender Netzwerk-Controller gefunden wurde. Der Tooltip des Icons gibt Auskunft über die Art der Fehlkonfiguration.

  • Service Type: Zeigt die Art des selektierten Netzwerk-Controllers an.

  • Name: Zeigt den Namen des selektierten Netzwerk-Controllers an.

  • Netzwerkname: Zeigt den Namen des Netzwerks an, in welchem sich der selektierte Netzwerk-Controller befindet.

Data / RPC Services

In der Tabelle Data / RPC Services sind die folgenden Spalten verfügbar:

  • S: Im Falle einer Fehlkonfiguration wird hier ein Icon angezeigt. Der Tooltip des Icons gibt Auskunft über die Art der Fehlkonfiguration.

  • Service Type: Zeigt die Art der Services an und ob diese bereitgestellt oder empfangen werden.

  • Topic / Funktionsname: Zeigt den Topicnamen bei Publish/Subscribe- bzw. den Funktionsnamen bei RPC-Services an.

  • Medientyp: Zeigt den für einen Publish/Subscribe- bzw. RPC-Service definierten Medientyp an.

  • Labels: Zeigt die für einen Publish/Subscribe- bzw. RPC-Service definierten Labels (Schlüssel-Wert-Paare) an.

[8] Seitenverwaltung

Klicken Sie auf den Pfeil Elemente pro Seite, um auszuwählen, wie viele Elemente pro Seite angezeigt werden sollen.

Um durch die Seiten zu blättern, verwenden Sie die Navigations-Icons.
